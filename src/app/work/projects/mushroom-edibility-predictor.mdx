## Overview

An end-to-end machine learning application designed to predict mushroom edibility with high accuracy and interpretability. The project emphasizes risk-aware classification and model explainability, crucial for applications where incorrect predictions could have serious consequences.

## Key Features

- **Random Forest Classifier**: Implemented robust ML model for binary classification (edible vs. poisonous)
- **Model Evaluation**: Comprehensive evaluation using Confusion Matrix and ROC Curve analysis
- **Feature Importance**: Detailed feature importance analysis for model interpretability
- **Risk-Aware Approach**: Prioritized precision and recall metrics to minimize false negatives
- **Streamlit Deployment**: Interactive web interface for real-time predictions
- **Explainability**: Built-in visualization tools to understand model decision-making

## Technologies Used

- **Python**: Core programming language for data processing and model development
- **Scikit-learn**: Machine learning library for Random Forest implementation and evaluation
- **Streamlit**: Framework for interactive web application deployment
- **Pandas**: Data manipulation and analysis
- **NumPy**: Numerical computing for data processing
- **Matplotlib/Seaborn**: Data visualization for model evaluation and feature analysis

## Challenges and Learnings

The primary challenge was balancing model accuracy with interpretability. In mushroom classification, false negatives (classifying poisonous mushrooms as edible) are far more dangerous than false positives. I learned to optimize for precision and implemented comprehensive evaluation metrics to ensure the model's reliability in real-world scenarios.

## Outcome

The Mushroom Edibility Predictor achieves high accuracy while maintaining strong interpretability. The application successfully demonstrates the importance of risk-aware ML development and provides a template for building responsible AI systems in domains where classification errors have significant consequences.
